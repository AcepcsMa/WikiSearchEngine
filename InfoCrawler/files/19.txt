				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Vector space model</b> or <b>term vector model</b> is an algebraic model for representing text documents (and any objects, in general) as <a href="/wiki/Vector_space" title="Vector space">vectors</a> of identifiers, such as, for example, index terms. It is used in <a href="/wiki/Information_filtering" class="mw-redirect" title="Information filtering">information filtering</a>, <a href="/wiki/Information_retrieval" title="Information retrieval">information retrieval</a>, <a href="/wiki/Index_(search_engine)" class="mw-redirect" title="Index (search engine)">indexing</a> and relevancy rankings. Its first use was in the <a href="/wiki/SMART_Information_Retrieval_System" title="SMART Information Retrieval System">SMART Information Retrieval System</a>.</p>

<p>Each <a href="/wiki/Dimension_(vector_space)" title="Dimension (vector space)">dimension</a> corresponds to a separate term. If a term occurs in the document, its value in the vector is non-zero. Several different ways of computing these values, also known as (term) weights, have been developed. One of the best known schemes is <a href="/wiki/Tf-idf" class="mw-redirect" title="Tf-idf">tf-idf</a> weighting (see the example below).</p>
<p>The definition of <i>term</i> depends on the application. Typically terms are single words, <a href="/wiki/Keyword_(linguistics)" title="Keyword (linguistics)">keywords</a>, or longer phrases. If words are chosen to be the terms, the dimensionality of the vector is the number of words in the vocabulary (the number of distinct words occurring in the <a href="/wiki/Text_corpus" title="Text corpus">corpus</a>).</p>
<p>Vector operations can be used to compare documents with queries.</p>

See <a href="/wiki/Cosine_similarity" title="Cosine similarity">cosine similarity</a> for further information.</p>

<h2><span class="mw-headline" id="Advantages">Advantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vector_space_model&amp;action=edit&amp;section=4" title="Edit section: Advantages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The vector space model has the following advantages over the <a href="/wiki/Standard_Boolean_model" title="Standard Boolean model">Standard Boolean model</a>:</p>
<ol>
<li>Simple model based on linear algebra</li>
<li>Term weights not binary</li>
<li>Allows computing a continuous degree of similarity between queries and documents</li>
<li>Allows ranking documents according to their possible relevance</li>
<li>Allows partial matching</li>
</ol>
<p>Most of these advantages are a consequence of the difference in the density of the document collection representation between Boolean and tf-idf approaches. When using Boolean weights, any document lies in a vertex in a n-dimensional <a href="/wiki/Hypercube" title="Hypercube">hypercube</a>. Therefore, the possible document representations are <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >
  <semantics>
    <mrow class="MJX-TeXAtom-ORD">
      <mstyle displaystyle="true" scriptlevel="0">
        <msup>
          <mn>2</mn>
          <mrow class="MJX-TeXAtom-ORD">
            <mi>n</mi>
          </mrow>
        </msup>
      </mstyle>
    </mrow>
    <annotation encoding="application/x-tex">{\displaystyle 2^{n}}</annotation>
  </semantics>
</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/8226f30650ee4fe4e640c6d2798127e80e9c160d" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -0.338ex; width:2.399ex; height:2.343ex;" alt="2^{n}" /></span> and the maximum Euclidean distance between pairs is <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >
  <semantics>
    <mrow class="MJX-TeXAtom-ORD">
      <mstyle displaystyle="true" scriptlevel="0">
        <mrow class="MJX-TeXAtom-ORD">
          <msqrt>
            <mi>n</mi>
          </msqrt>
        </mrow>
      </mstyle>
    </mrow>
    <annotation encoding="application/x-tex">{\displaystyle {\sqrt {n}}}</annotation>
  </semantics>
</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/2a2994734eae382ce30100fb17b9447fd8e99f81" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -1.005ex; width:3.351ex; height:3.009ex;" alt="{\sqrt {n}}" /></span>. As documents are added to the document collection, the region defined by the hypercube's vertices become more populated and hence denser. Unlike Boolean, when a document is added using tf-idf weights, the idfs of the terms in the new document decrease while that of the remaining terms increase. In average, as documents are added, the region where documents lie expands regulating the density of the entire collection representation. This behavior models the original motivation of Salton and his colleagues that a document collection represented in a low density region could yield better retrieval results.</p>
<h2><span class="mw-headline" id="Limitations">Limitations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Vector_space_model&amp;action=edit&amp;section=5" title="Edit section: Limitations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The vector space model has the following limitations:</p>
<ol>
<li>Long documents are poorly represented because they have poor similarity values (a small <a href="/wiki/Scalar_product" class="mw-redirect" title="Scalar product">scalar product</a> and a <a href="/wiki/Curse_of_dimensionality" title="Curse of dimensionality">large dimensionality</a>)</li>
<li>Search keywords must precisely match document terms; word <a href="/wiki/Substring" title="Substring">substrings</a> might result in a "<a href="/wiki/False_positive" class="mw-redirect" title="False positive">false positive</a> match"</li>
<li>Semantic sensitivity; documents with similar context but different term vocabulary won't be associated, resulting in a "<a href="/wiki/False_negative" class="mw-redirect" title="False negative">false negative</a> match".</li>
<li>The order in which the terms appear in the document is lost in the vector space representation.</li>
<li>Theoretically assumed terms are statistically independent.</li>
<li>Weighting is intuitive but not very formal.</li>
</ol>
<p>Many of these difficulties can, however, be overcome by the integration of various tools, including mathematical techniques such as <a href="/wiki/Singular_value_decomposition" class="mw-redirect" title="Singular value decomposition">singular value decomposition</a> and <a href="/wiki/Lexical_database" title="Lexical database">lexical databases</a> such as <a href="/wiki/WordNet" title="WordNet">WordNet</a>.</p>
